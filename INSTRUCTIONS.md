# Project Instructions & Architecture

This document provides an overview of how the East Africa Bridge Group platform works and how to set up the environment, particularly for the Super Admin.

## Technology Stack

- **Framework**: Next.js 16 (App Router)
- **CMS**: Payload CMS 3.85 (Headless)
- **ORM**: Prisma 7
- **Database**: PostgreSQL
- **Styling**: TailwindCSS v4 & Shadcn UI
- **Language**: TypeScript

## How the App Works

The application combines the power of **Payload CMS** for content and user management with **Prisma** for custom data modeling and high-performance queries.

### 1. Payload CMS Integration
Payload CMS handles:
- **Authentication**: User roles (Super Admin, Admin, Editor, Content Manager) and sessions.
- **Collections**: Media, Commodities, Categories, Posts (Blog), Documents, and Activity Logs.
- **Mini CRM**: Contact submissions and in-app notifications.
- **Globals**: Dynamic configuration like SMTP Email Settings.
- **Localization**: Support for English and Arabic content.

### 2. Prisma ORM
Prisma is used to:
- Define the database schema that aligns with Payload CMS tables.
- Perform custom database operations and seeding.
- Provide a typed client for non-CMS managed data (like certain Page structures).

## Initial Setup & Super Admin Login

To set up the initial Super Admin user and populate the database with required schema and data, follow these steps:

### 1. Database Configuration
Ensure your `DATABASE_URL` and `PAYLOAD_SECRET` are set in your environment or `.env` file.

### 2. Database Seeding (Crucial)
Run the following command to initialize the database schema and create the Super Admin user:

```bash
npm run seed
```

This command executes `npx prisma db seed`, which triggers the `prisma/seed.ts` script. This script uses the **Payload Local API** to:
- Create the **Super Admin** account.
- Initialize core collections (Categories, Commodities).
- Setup global Email Settings.

### 3. Super Admin Credentials
Once seeded, use these credentials to log in at `/login`:

- **URL**: `http://localhost:3000/login`
- **Email**: `admin@eabridgegroup.com`
- **Password**: `Admin@1234`

### 4. Database Schema Stability
The project is configured to ensure that Payload CMS and Prisma work harmoniously. The `prisma/schema.prisma` file includes exhaustive mappings (`@@map`) for all tables generated by Payload.

**Important**: Always ensure that any changes to Payload collections are reflected in the `prisma/schema.prisma` file to prevent Prisma from dropping tables during synchronization.

## Common Commands

- `npm run dev`: Start the development server (uses `--webpack` for compatibility).
- `npm run build`: Build for production.
- `npm run seed`: Initialize/reset seed data and Super Admin.
- `npx prisma validate`: Check schema integrity.
